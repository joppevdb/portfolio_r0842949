SDM630

This is an independent project that I started in my spare time. Currently the basis of the project is already working, but I am still working on extensions to make make it more user-friendly.

The end goal of the project is to be able to switch different circuits in a house according to the imports or exports measured on the SDM630 meter.

Technical explanation

The more technical process goes as follows. The meter is read out using a raspberry pi. This pi will retrieve the information on the basis of a modbus. The retrieved information is then written to the database (also running on the raspberry pi). After this, the results are studied and the circuits are switched on or off where necessary (by setting a pin high or low on the pi). All this information is also shown on a webpage using an API. web page.

Technologies: Python, Javascript, HTML and CSS

Soft skills: Self-motivation, Problem solving

Hard skills: Reading modbus, Connecting database with API

PROJECTWEEK

The project week is part of the of System analysis. During this project week, we are given the task of building a smart coffee machine. This smart coffee machine would then have to take the place of the coffee machines that are currently used on campus.

Beforehand, we had to spend half a year in a group of 4 planning everything around what exactly we were going to build. During these six months we had to make several reports and perform different analyses.

During the project week itself, everything was built. The coffee machine itself was simulated by a raspberry pi with different push buttons and lights. Then a front-end was set up to control the raspberry pi. Through api-calls the database was kept up to date. Finally, all the information was displayed on a dashboard that could be operated by the administrator.

I mainly worked on the front-end and the api-calls that retrieve information from the database.

Technologies: Python, Javascript, HTML, CSS, Java and Grafana

Soft skills: Teamwork, Responsibility, Communication, Decisiveness

Hard skills: Create api-call in java, Style web page, Make web page run a python script

API

For the final project of the course API development, we were asked to create an API connected to a database (SQL). This API should be built completely automatically with GitHub actions. After this, the API will be hosted on Okteto and connected to a front-end hosted on Netlify.

Thema

The chosen theme is beer because I personally find it an interesting theme. The chosen theme is beer. The API is built around tracking which brewery has brewed which beers.

API

The API is more complex to keep an overview the API is divided into three parts (beers, breweries, owners). For each part, a table is created in the database. Then a post is created for each component so that data can be added quickly. Finally, for each section there are also different gets to retrieve the data. For the beers component, a delete is also added.

Technologies: Docker, Fastapi, MariaDB, Github

Soft skills: Self-motivation, Problem-solving, Creativity

Hard skills: Github actions, Making own docker-compose, Creating API-calls
